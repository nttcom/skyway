Alpha release of multi-user connections and SFU functionality

August 8, 2016

We have added a new API to SkyWay that lets you use a multi-user room API and SFU functionality. You can now connect to multiple users in about 10 lines of code.

New Features

Multi-user connections (Room API)

Create group video chats easily with out new Room API! Just specify the room name and connect to all users in the room. Of course we still support 1 on 1 connections.

SFU

WebRTC is in generally a technology that connections P2P without using any servers. SkyWay's existing API is based on communication in a P2P fashion.

SFU(Selective Forwarding Unit) is a technology that enables you to broadcast media and data via a server to multiple peers, instead of using connecting P2P.

There are 2 ways of using this new API, SFU and full mesh. Using the SFU mode relays all communication through SkyWay's servers. Since clients only have to send their data to one endpoint, it reduces stress on the clients, allowing more simultaneous connections.

Using full mesh mode connects to all users in the room using P2P connections.
